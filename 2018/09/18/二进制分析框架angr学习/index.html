二进制分析框架angr学习

2018-09-18 PV:

学习angr，又是不停地挖坑填坑。

简介

  • angr是新一代二进制文件分析框架，旨在解决下列问题：
    • 载入二进制文件到分析程序中；
    • 将二进制文件翻译成中间表示；
    • 执行真正的分析，可能是：
      • 一部分或全程序的静态分析；
      • 一种程序状态空间的符号搜索；
    • 以上方法的部分结合使用。
  • 二进制的载入：CLE
  • 中间表示：VEX（使angr实现跨底层结构分析二进制文件）
  • 求解程序引擎：约束求解和其它计算需求由angr子模块Claripy提供
  • 程序状态集：angr提供仿真机器状态的接口。理解这是成功使用angr的必要。
  • 程序路径：程序可以由执行过程可能经过的路径进行分析。angr显示了路径执行过程中的信息。
  • 语义表示：angr一个强大的特性展示基本块及其在程序状态中的影响。angr可以说明基本块的作用，该功能由SimuVEX子模块实现。
  • 符号执行：angr提供符号执行引擎。及该引擎的接口，以及如何使用。
  • 全程序分析：以上所有组件结合起来进行复杂的，全程序的分析。

常用模块

Surveyors

通过surveyors下的方法Explorer来进行符号执行。参数find为要找到的地址，avoid为避免遇到的地址。最后一行中的se为solver engine的意思。

Path group

最常用的方法。

通过blank_state设定起始点，然后将状态s作为参数放入path_group，然后和surveyors类似，进行explore。

Symbolic args

从angr放置符号参数。

通过claripy库，设定一个BVS参数，名字为args，长度为8*16bit。

Symbolic input

如果通过gets()或scanf()读入参数，通过另外两种方法放置：

放在posix上

先将每posix的每个限制加入posix的每个字里面。后面的posix.files[0]就是stdin。

直接放在记忆体位置上

输入完后将输入的值存在一个记忆体位置上（输入地址存在执行地址之前），符号执行时直接找到记忆体地址。存储时遵循小端序，所以要调用reverse。

Breakpoint

在符号执行遇到记忆体写入的时候，下断点即可。

Hook

不想让符号执行在某些不必要的函数处浪费时间的话，可以用hook把函数勾住。这样在符号执行时，会执行上面set_rax的指令。

常用函数或方法

函数-Project

angr脚本中必用的一个函数，用来载入一个二进制文件。一般的形式：p = angr.Project('./vul', load_options={'auto_load_libs': False})。相关参数：

  • thing：要分析的执行路径
  • default_analysis_mode=None：预设的分析模式
  • ignore_functions=None：选择跳过分析的函数名
  • use_sim_procedures=True：是否使用已经预先建立好的模拟程序
  • exclude_sim_procedures_func=None：排除不要的模拟程序
  • exclude_sim_procedures_list=()：以元组的形式排除不要的模拟程序
  • arch=None：设定执行的结构
  • simos=None：自定模拟系统的类别
  • load_options=None：CLE loader的参数名字
  • translation_cache=False：翻译基本区块时进行快取
  • support_selfmodifying_code=False：是否支援修改自我程式

方法-path_group

相关方法：

  • explore：大致设定找路径时的stash，将参数送至step
  • step：让每条路径都走一步
  • _one_step：定义每一步做什么
  • _one_path_step：检查路径是否出错
  • _record_step_results：记录并重新归类路径

相关参数：

  • project：即载入的二进制文件
  • active_paths=None：设置初始状态
  • stashes=None：种类有——'active'、'stashed'、'pruned'、'unsat'、'errored'、'deadended'、'unconstrained'
  • hierarchy=None：记录路径的前后节点
  • veritesting=None：符号执行卡住时，可以用这个跑完
  • veritesting_options=None：veritesting的参数设置
  • immutable=None：是否备份路径
  • resilience=None：是否忽略异常
  • save_unconstrained=None：是否记录并保留不受限制的路径（寻找缓冲区溢出的漏洞）
  • save_unsat=None：是否记录并保留不符合的路径
  • strong_path_mapping=None：PathHierarchy()函数中的参数，使用weakref.WeakValueDictionary()或{}
  • threads=None：是否使用多线程

explore相关参数：

  • stash=None：指定stash的种类
  • n=None：选择让符号执行执行几回合
  • find=None：想找到的指令地址
  • avoid=None：想避免遇到的指令地址
  • num_find=None：要找到的目标路径
  • found_stash=None：选择找到目标路径后存放的stash
  • avoid_stash=None：选择找到规避路径后存放的stash
