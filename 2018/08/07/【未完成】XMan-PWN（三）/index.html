WHAT is _IO_FILE

_IO_FILE structures

  • All the FILE structures will be linked in a linked list
    • The head is the _IO_LIST_ALL, we can go through all the FILE structures with this linked list head
  • Three FILE structures will be opened automatically
    • stdin, stdout, stderr
    • saved in the rw- section of libc in the memory

struct _IO_FILE *_chain

_IO_FILE_plus structures

vtable is a _IO_jump_t pointer, some std I/O functions will use these pointers.

  • fread
  • fwrite
  • fopen
  • fclose
  • printf/puts

fread

  • buffer => save data
  • size => size of each object
  • count => how many objects
  • stream => file stream
  • return value => readed objects

fwrite

  • buffer => write data
  • size => size of each object
  • counts => num of objects
  • stream => file stream pointer
  • return value => num of written objects

fopen

  • filename => target file
  • type => open type(r/w/x)
  • return value => file pointer

fclose

Close a file stream, output the rest of data in the buffer area to the drive

  1. unlink the fp from the linked list
  2. call _IO_file_close_it to close the fp
  3. _IO_FINISH(fp)

printf/puts

  • _IO_puts
  • _IO_sputn
  • _IO_new_file_xsputn
  • write
