将程序拖进OD后，找到main函数。

阅读汇编程序，推测源程序的代码

这里可以看到puts函数输出一段字符串：“please input the flag”，然后有一个scanf来接收输入的字符串。

关键代码段处：

这里的[local.5]可以理解为一个变量，EAX中存放的是变量[local.5]的值，EDX中存放的是输入的字符串中的一个字符。设[local.5]的值为i，每次取出字符串中第i个字符，将该字符与i进行异或，结果存入字符串的第i个。

容易知道需要求出解密算法才能获得正确的字符串

因为异或运算可以通过二次运算重新得到

故这里的加密属于一种对称加密方式，只需要将被加密的字符串再加密一次就能的到

拿到flag：

Easy_CrackMe.exe

拿到程序后，先进行简单的测试

可以判断出输入的字符串应该有一个函数接收，即GetDigItemTextA。找到之后，慢慢往下读

“byte ptr ss:[esp+0x5]”表示的是输入字符串的某一位置的字符。这里需要与字母a（a的ASCII码为97，即0x61）进行比较，可以说明flag中含有E。

下面的“push Easy_Cra.00406078”后有注释“5y”，那么flag中应有这一片段

这里先将“R3versing”字符串存入ESI中，然后EAX取输入字符串的一部分。每次取两个字符串分别比较，比完后再向后移两位。那么可以判断“R3versing”也应该是flag的一部分

最后发现进行了最后一次比较，在“byte ptr ss:[esp+0x4]”处，显然在字符串的最前面。该处应为字母E（E的ASCII码为69，即0x45）

最后可以得出flag应为以上几个部分按顺序拼接后的结果
