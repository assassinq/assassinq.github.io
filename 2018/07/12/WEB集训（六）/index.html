跑目录&文件泄露。

跑目录（AWVS使用教程）

ACUNETIX WEB VULNERABILITY （简称AWVS）是一款知名的商业Web网络漏洞扫描工具，它通过网络爬虫测试你的网站安全，检测流行安全漏洞。

具体的界面是这样的：

Site Crawler

在中间输入url地址如http://192.168.44.128/dvwa/，点击最右侧的绿色start。就能通过链接等各种方式爬网。

如果是需要身份认证才能访问的内容，则在右侧的login sequence设置，选择一个事先保存的登录过程。

Target Finder

输入ip地址段和端口列表，先扫描ip端口，找到目标端口后再扫描端口上的服务。

Subdomain Scanner

用DNS进行域名解析，列出它的子域名和主机名。这样以后可以进行针对性的扫描。

Webscan

  • File——New——Web Site Scan：新建一次网站扫描
  • File——New——Web Site Crawl：新建一次网站爬行
  • File——New——Web Services Scan：新建一个WSDL扫描
  • Load Scan Results：加载一个扫描结果
  • Sava Scan Results：保存一个扫描结果

文件泄露

文件泄露, 根据泄漏的信息敏感程度, 在WEB漏洞中可以算是中危甚至高危的漏洞。

Git

Git是一个开源的分布式版本控制系统。

功能

  1. 从服务器上克隆完整的Git仓库（包括代码和版本信息）到单机上。
  2. 在自己的机器上根据不同的开发目的，创建分支，修改代码。
  3. 在单机上自己创建的分支上提交代码。
  4. 在单机上合并分支。

git在运行git init初始化项目时, 会在项目的根目录(可用git rev-parse –show-toplevel查看)创建一个.git的隐藏文件夹，来记录代码的变更等历史记录。如果将这个目录置于WEB的路径下让用户可以访问，那么也就泄露了几乎所有的源代码和其他敏感信息。

危害（泄露内容）

  • 所有该项目的源代码
  • 私有仓库的地址
  • 私密的配置信息
  • 所有commiter的邮箱帐号信息
  • (可能)内部的帐号和密码

利用方法

常规的利用方法就是下载整个目录, 然后用git命令回滚整个项目：

下载.git文件：wget -r --no-parent --mirror http://www.example.com/.git cd www.example.com

代码重构：git reset –hard

相关的自动化利用脚本

  • dvcs-ripper：基于perl的工具,除了git还支持很多版本管理系统（如SVN）

rip-git.pl -v -u http://www.xxx.com/.git/

  • GitHack：.git泄露利用脚本，通过泄露的.git文件夹下的文件，还原重建工程源代码。

GitHack.py http://www.xxx.com/.git/

备份文件泄漏

有两种情况：

  1. 运维人员偷懒地直接在网站根目录用类似tar -czvf bakup.tgz *的命令将网站进行备份，这样整站的源代码都能直接被用户打包下载了。
  2. 开发或者运维人员使用的编辑器修改文件时自动备份了所编辑的网页内容，从而泄露了该网页的源代码。

具体泄漏文件：.rar、.zip、.7z、.tar.gz、.bak、.swp、.txt、.html、~。

PS：关于.swp，vim中的swp即swap文件，在编辑文件时产生的隐藏文件，如果原文件名是data，那么swp文件名就是.data.swp。如果文件正常退出，则此文件自动删除。

如果vim非正常退出，除非手动删除swp文件，否则它会一直存在。同时，再次编辑的时候也会有提示“……swp already exists”
多个程序同时编辑一个文件，swp文件也不会删除。

可以通过选项禁止产生swp文件
/etc/vimrc并添加set noswapfile set noswapfile。

PS：index.php~也就是index.php.swp。

利用方法

  • 对于打包文件而言, 渗透测试人员可以用{常用文件名}+{常用压缩包后缀}的方式扫描网站。
  • 对于网页的临时备份文件, 可以扫描对应页面的.swp或者.bak等后缀, 说不定也能找到有用的信息。

.DS_Store

Desktop Services Store，是macOS目录下的隐藏文件，包含了当前目录结构和一些的自定义信息，如背景和图标位置等, 在windows下类似的文件desktop.ini。暴露了.DS_Store文件也就相当于暴露了该目录下的所有内容。

.DS_Store的格式为二进制, 内部数据结构为Proprietary格式，可以自行解析并递归下载所有文件。

WEB-INF泄漏

在Java的Servlet文档中，说到WEB-INF目录“包含了所有web应用会用到但是不处于web路径中的资源”，也就是说，WEB-INF目录下的内容是不属于公开页面的。
 web应用可以通过getResource等API在servlet的上下文中访问到这些资源。

通常开发者会把许多JSP文件、Jar包、Java的类文件放在该目录下。一般目录的内容都是可以预测的：

  • WEB-INF/web.xml：Web应用程序配置文件，描述了servlet和其他的应用组件配置及命名规则。
  • WEB-INF/database.properties：数据库配置文件。
  • WEB-INF/classes/：一般用来存放Java类文件（.class）。
  • WEB-INF/lib/：用来存放打包好的库（.jar）。
  • WEB-INF/src/：用来放源代码（.asp和.php等）。

利用方法

通过web.xml文件推测应用组件相关类的名字, 然后在src目录下查找代码, 如果没有源代码可以直接下载class文件反编译即可。
